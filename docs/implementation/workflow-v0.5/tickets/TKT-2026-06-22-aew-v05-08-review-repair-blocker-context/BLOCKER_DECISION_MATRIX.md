# Blocker Decision Matrix

| Scenario | Authorization | Required Result |
| --- | --- | --- |
| Focused in-scope repair inside accepted plan | preapproved | Return exact edit boundary and validation. |
| Dependency install with accepted plan, existing package manager, lockfile, reviewer evidence, and tests | preapproved | Return exact command, validation, and rollback. |
| Dependency change not accepted by plan | user_required | Ask one approval question. |
| Reversible local/test migration against disposable state | preapproved | Return command, validation, and rollback. |
| Irreversible or persistent migration | user_required | Ask one approval question. |
| Cleanup of artifacts generated by active run | preapproved | Return exact paths and validation. |
| Cleanup of user work or unrelated files | user_required | Ask one approval question. |
| Scoped commit/push assigned by ticket delivery | preapproved | Use explicit staging only and prove upstream equality. |
| Force push, remote history rewrite, branch deletion, deploy, release, publish, or external side effect | user_required | Ask one approval question. |
| Secrets, credentials, `.env*`, or private config | user_required | Do not read or modify secret values. |
| Irreducible product, policy, scope, or acceptance decision | user_required | Ask one concrete decision question. |
| Unsafe or under-specified request | user_required | Classify as unsafe or decision-required before action. |
